# PROJECT MANIFESTO: ALGORA

## 1. Identity & Philosophy

**Name:** ALGORA
**Etymology:** **ALGO**rithm (Computational Logic) + **AGORA** (Ancient Greek Marketplace & Assembly).
**Core Concept:** "The Digital Agora of Intelligence."
**Philosophy:** В мире, утонувшем в информационном шуме, дефицита данных не существует — существует дефицит ясности. Algora строит систему ИИ-агентов, которые фильтруют весь интернет и выдают концентрат: короткий, точный, полезный факт для конкретной аудитории. Без воды. Без мусора. Только то, что влияет на решения.

Algora продаёт не информацию, а **Ясность (Clarity)** и **Связи (Connections)**.

## 2. Vision & Mission

**Vision:** Экосистема взаимосвязанных аудиторий, где каждая группа решает проблемы другой — объединённые на единой платформе через доверие, построенное на ценном контенте.

**Mission:** Создать модульную, полностью автоматизированную систему генерации целевых аудиторий через ИИ-контент, а затем соединить эти аудитории на платформе для взаимного решения потребностей.

**Mechanism:** ИИ-агенты мониторят источники информации, фильтруют шум, извлекают факты и доставляют их целевым аудиториям в формате концентрированной ценности.

## 3. Strategic Architecture (3 фазы)

Проект строится последовательно. Каждая фаза — фундамент для следующей.

### Фаза 1: Audience Engine (Единица генерации аудитории)
Создать один полностью автоматизированный инструмент, который:
- Мониторит источники информации по выбранной нише
- Фильтрует шум, извлекает факты
- Генерирует контент-концентрат (короткий факт + контекст + actionable insight)
- Публикует в канал (Telegram / другие каналы)
- Наращивает целевую аудиторию органически

**Результат фазы:** Работающий прототип + растущая аудитория в одном сегменте.

### Фаза 2: Replication (Масштабирование)
Копировать инструмент и перенастраивать на другие целевые сегменты:
- Каждый новый сегмент выбирается так, чтобы его потребности были связаны с уже набранными аудиториями
- Одна категория может решить проблемы другой категории
- Промежуточная монетизация: продажа рекламы на набранную аудиторию (не основная цель)

**Результат фазы:** Несколько целевых аудиторий с пересекающимися потребностями.

### Фаза 3: Platform (Соединение)
После набора критической массы — создать платформу, где аудитории соединяются:
- Формат платформы определяется на основе данных о реальных потребностях аудиторий
- Монетизация через транзакции, подписки, premium-доступ

**Результат фазы:** Работающая платформа с доходом.

## 4. Текущий этап: Исследование рынка

Прежде чем строить первую "Единицу генерации аудитории", необходимо:

### 4.1 Задачи исследования
1. Идентифицировать реальную боль конкретного целевого сегмента
2. Определить, может ли ИИ генерировать для этого сегмента контент реальной ценности
3. Проверить, существует ли второй сегмент, который решает проблемы первого
4. Оценить потенциал связки "сегмент A ↔ сегмент B" для будущей платформы

### 4.2 Географии
Преимущества разных рынков и кросс-граничные возможности:
- **Россия** — дефицит западных товаров/технологий, растущий внутренний рынок, потребность в альтернативных поставщиках
- **Китай** — мировая фабрика, избыток производственных мощностей, ищет новые рынки сбыта
- **Индия** — быстрорастущий IT-сектор, дешёвая квалифицированная рабочая сила, огромный внутренний рынок
- **Турция** — логистический хаб между Европой и Азией, текстиль, стройматериалы, медицинский туризм
- **Европа** — высокая покупательная способность, спрос на качественные услуги и нишевые товары

### 4.3 Критерии выбора первого сегмента

При выборе сегмента для первой "Единицы" оцениваем по 7 критериям:

| Критерий | Описание | Вес |
|----------|----------|-----|
| **Острота боли** | Насколько критична проблема? Теряет ли аудитория деньги/время без решения? | Высокий |
| **Фактоёмкость** | Можно ли решить боль фактами и аналитикой (а не эмоциональным контентом)? | Высокий |
| **Доступность источников** | Есть ли открытые источники данных, которые ИИ может парсить? | Высокий |
| **Размер аудитории** | Достаточно ли людей в этом сегменте для критической массы? | Средний |
| **Платёжеспособность** | Готова ли аудитория платить (сейчас или в будущем на платформе)? | Средний |
| **Связанность с другими сегментами** | Есть ли естественный "парный" сегмент для будущей платформы? | Средний |
| **Конкуренция** | Насколько насыщен рынок аналогичными решениями? | Низкий* |

*Низкий вес конкуренции, потому что если боль острая и решение качественное — конкуренция преодолима.
